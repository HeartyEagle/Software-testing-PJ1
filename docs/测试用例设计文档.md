# 测试用例设计文档

项目名称：`SoftwareTestingDemo` 体育场馆预约管理系统  
文档日期：`2026-04-21`  
测试框架：`JUnit 5 + Mockito + Spring Boot Test + MockMvc + H2`

## 0. 测试设计技术与覆盖策略

### 0.1 黑盒测试设计

本项目黑盒部分主要采用等价类划分与边界值分析。

- 等价类划分
  - 合法等价类：已登录用户访问个人功能、管理员访问后台功能、存在的 `orderID` / `messageID` / `newsID` / `venueID`、合法分页参数 `page>=1`
  - 非法等价类：未登录访问受保护接口、普通用户访问管理员接口、操作他人资源、缺少必填参数、查询不存在对象、上传空文件名或非法文件名
- 边界值分析
  - 分页边界：`page=1`、缺省 `page`、非法 `page=0`
  - 文本输入边界：空内容、正常内容、异常身份参数
  - 文件输入边界：空文件、带后缀文件、不带后缀文件
  - 日期时间边界：`date` 与 `startTime` 一致、`date` 与 `startTime` 不一致、场馆不存在时按日期查询

### 0.2 白盒测试设计

白盒部分主要针对 Service 层与部分控制器关键分支，采用语句覆盖与判定覆盖。

- 语句覆盖
  - 覆盖新增、删除、修改、分页查询、VO 聚合等核心语句
- 判定覆盖
  - 覆盖对象存在 / 不存在
  - 覆盖已登录 / 未登录
  - 覆盖本人操作 / 越权操作
  - 覆盖管理员 / 非管理员
  - 覆盖空文件 / 非空文件 / 无后缀文件

### 0.3 缺陷导向补测策略

在原有“正常流程可用性”测试基础上，本轮补充了一组缺陷导向测试，重点验证以下高风险场景：

- 未鉴权接口可被匿名直接调用
- IDOR 越权：用户可操作他人订单、留言、资料
- 参数被忽略导致业务行为与表单不一致
- 空值 / 非法输入导致 NPE 或字符串越界异常

## 三、说明

本文件已按“测试对象 + 测试脚本 + 逐条测试样例表格”的方式覆盖当前测试脚本中的主要 `@Test` 用例，包含：

- `service.impl` 单元测试
- 首页与 User 侧 Controller 集成测试
- Admin 侧 Controller 集成测试

截至当前版本，测试结果分为两类：

- 原有基础功能测试多数通过，用于证明正常业务流程可运行
- 新增缺陷导向测试中已有多项失败，这些失败并非测试脚本错误，而是用于暴露当前系统中的真实缺陷，包括未鉴权访问、IDOR 越权以及参数被忽略等问题

因此，本项目当前测试结论不再是“全部通过”，而是“基础流程通过，但已通过新增测试定位出多项高风险缺陷，需在修复后重新执行回归测试”。
